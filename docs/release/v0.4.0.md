# iqdb-types v0.4.0 — Feature Freeze

**The surface is complete.** v0.4.0 declares the public type set of `iqdb-types` **frozen**: no new types or methods will be added before 1.0. What earns the freeze is proof rather than assertion — a consumer-simulation suite assembles working analogues of the three real downstream crates (`iqdb-distance`, an index crate, and `iqdb-filter`) using only the public surface, at the exact signatures those crates expose. If a working vector database can be built from this vocabulary alone, the vocabulary is done. From here, only the deliberate v0.5.0 API freeze and the RC polish remain.

## What is iqdb-types?

The root of the iQDB dependency graph — the shared vocabulary every other `iqdb-*` crate speaks: `Vector`, `VectorId`, `DistanceMetric`, `Metadata`/`Value`, `Filter`, `SearchParams`, `Hit`, and the `IqdbError` domain error. It is pure data: no engine, no storage, no I/O, and a single runtime dependency (`error-forge`). Because a breaking change here cascades through the whole family, the surface is reviewed hard before each freeze.

## What's new in 0.4.0

### Consumer-simulation suite — the feature-freeze gate

The ROADMAP gates the feature freeze on *"at least two downstream iQDB crates compile against this unchanged."* Those crates live outside this repository, so `tests/consumer_simulation.rs` builds minimal **working analogues** of all three real consumers from the public surface alone — and at the **exact signatures the real Cortex crates expose**, cross-checked against those implementations and their roadmaps:

- **`iqdb-distance`** — `compute(metric: DistanceMetric, a: &[f32], b: &[f32]) -> Result<f32>`: all five metrics over raw slices, with the `#[non_exhaustive]` wildcard arm a real consumer must write. Verified against `dispatch::compute`.
- **An index crate** (`iqdb-flat`-shaped) — an `Arc<[f32]>`-storing index with `insert(VectorId, Arc<[f32]>, Option<Metadata>)`, `search(&[f32], &SearchParams) -> Result<Vec<Hit>>`, and `delete(&VectorId)`; dimension/duplicate/metric/not-found error paths included. Verified against the `IndexCore` trait.
- **`iqdb-filter`** — closed-world `Filter`-tree evaluation over `Metadata`, proving `Value` ordering is implementable from the public surface (even though `Value` is only `PartialEq`).

An end-to-end test wires all three together — a filtered top-`k` cosine search returning ranked `Hit`s with metadata — exercising the whole vocabulary the way the family will.

### Feature freeze declared

The public type set is complete. No new types or methods will be added before 1.0; subsequent 0.x releases are hardening, the API freeze, and RC polish only. The crate contains no `todo!`/`unimplemented!` and no `unsafe`.

## Compatibility note

`DistanceMetric` became `#[non_exhaustive]` in 0.3.0. The real `iqdb-distance::compute`/`compute_batch` currently match the five metrics **without** a wildcard arm; when that crate is built against `iqdb-types >= 0.3`, each match needs a `_ =>` arm (e.g. returning `IqdbError::InvalidMetric`). This is the intended, one-time cost of making the metric set forward-compatible.

## Breaking changes

**None.** No public API changed in this release; it adds tests and declares the existing surface frozen.

## Verification

Run on Windows x86_64, on **both** the stable toolchain (1.95.0) and the **1.87 MSRV** toolchain (`cargo +1.87`). The same commands run in the CI matrix across Linux, macOS, and Windows at stable and 1.87:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
cargo audit
```

All green on stable **and** 1.87. Test counts at this tag:

- **Default features:** 77 (0 unit + 35 integration + 42 doctests).
- **`--all-features`:** 88 (0 unit + 45 integration + 43 doctests).

The integration growth over 0.3.0 is the nine consumer-simulation tests. `cargo deny check` reports `advisories ok, bans ok, licenses ok, sources ok`; `cargo audit` finds zero advisories.

## What's next

- **v0.5.0 — API freeze (handle with care).** Final API review and a conservative lock of the public API for 1.x. The one open decision: whether `VectorRef` stays — the real consumers pass `&[f32]` directly, so its place is being re-examined. Cross-platform verification; `cargo audit` + `cargo deny` clean.
- **v0.6.0 → v0.9.x — Alpha / Beta / RC.** Bug-fixes and doc polish toward 1.0.
