# iqdb-types v0.3.0 ‚Äî Hardened & Documented

**The vocabulary, proven.** v0.3.0 adds no new public types ‚Äî instead it takes the v0.2.0 surface and makes it production-grade: every type contract is now checked against generated inputs with `proptest`, every public item ships a runnable example and a complete `docs/API.md` entry, the owned `Vector` is tuned for resident-memory workloads, and the metric enum is made forward-compatible so future metrics never force a breaking change. This is the release that turns "correct" into "trustworthy to build a family on."

## What is iqdb-types?

The root of the iQDB dependency graph ‚Äî the shared vocabulary every other `iqdb-*` crate speaks: `Vector`, `VectorId`, `DistanceMetric`, `Metadata`/`Value`, `Filter`, `SearchParams`, `Hit`, and the `IqdbError` domain error. It is pure data: no engine, no storage, no I/O, and a single runtime dependency (`error-forge`). Because a breaking change here cascades through the whole family, the surface is deliberately small and is reviewed hard before it freezes at `1.0`.

## What's new in 0.3.0

### Property-based test suite

`tests/properties.rs` (new `proptest` dev-dependency) checks the type contracts `dev/DIRECTIVES.md` ¬ß5/¬ß8 require to hold against *generated* inputs rather than hand-picked cases:

- **`Vector`** ‚Äî `new` accepts every non-empty, all-finite buffer and preserves it exactly; a non-finite component injected at any position is rejected with `InvalidVector`; `TryFrom` agrees with `new`.
- **`VectorRef`** ‚Äî every accessor reports the underlying slice faithfully.
- **`VectorId`** ‚Äî `U64` renders as decimal; `Bytes` renders as fixed-width lowercase hex that decodes back to the original bytes; `Debug` never collapses into `Display`.
- **`Metadata`** ‚Äî entries iterate in strict ascending key order and every key resolves to its value.
- **`DistanceMetric`** ‚Äî the `Eq`/`Hash` contract holds (equal values hash equal).
- **serde** ‚Äî a JSON round-trip (`from_str(to_string(v)) == v`) for **every** public type: `Value`, `Vector`, `VectorId`, `DistanceMetric`, `Metadata`, `Filter`, `Hit`, `SearchParams`.

### Runnable examples

A new `examples/` directory, each example documented and runnable:

- `vectors` ‚Äî owned/borrowed construction, validation, reclaiming the buffer.
- `metadata_and_filters` ‚Äî building metadata and filter trees, with the closed-world `neq` vs `not(eq)` distinction made concrete.
- `search` ‚Äî Tier-1 `SearchParams::new` and Tier-2 struct-update tuning; reading a `Hit`.
- `errors` ‚Äî `kind()` / `caption()` / `Display` and matching on the cause.
- `serde_roundtrip` ‚Äî every public type through JSON (gated behind `required-features = ["serde"]`).

### Complete API reference

`docs/API.md` is rewritten as an exhaustive reference: every public item with its description, parameters, trait/derive list, and **multiple worked examples per use case**, plus a trait-implementation matrix and the feature-flag table.

### Performance: `Vector` is now `Box<[f32]>`-backed

`Vector` stores its components in a `Box<[f32]>` rather than a `Vec<f32>`. Because a `Vector` is immutable after construction, it never needs spare capacity:

- the wrapper is one machine word smaller (16 vs 24 bytes), and
- the backing allocation is sized exactly to the data (no slack).

For workloads holding millions of vectors resident, that is real memory saved and a smaller, more cache-friendly value. **The public API is unchanged** ‚Äî `Vector::new` still takes a `Vec<f32>`, and `into_inner` still returns one via an allocation-free `Box<[f32]>::into_vec`.

Trivial accessors and constructors across the public surface are now `#[inline]`, so consumers inline them across crate boundaries even without enabling fat LTO.

### Forward compatibility

`DistanceMetric` is now `#[non_exhaustive]`, so a future metric (Jaccard, Chebyshev, ‚Ä¶) can be added without a breaking change. Consumers matching on it must include a wildcard arm. (`IqdbError` was already `#[non_exhaustive]`; `Value` and `Filter` remain exhaustive so they can be matched fully.)

### Benchmark suite

`benches/vector_new.rs` (new `criterion` dev-dependency) measures the one compute-bearing path in the crate ‚Äî `Vector::new` validation plus the shrink-to-`Box<[f32]>` ‚Äî across common embedding dimensionalities (32 / 128 / 768 / 1024), with throughput reported per element. Run with `cargo bench --bench vector_new`; reports land in `target/criterion/`.

## Breaking changes

**None to the public API.** The `Vector` representation change is internal (its constructors and accessors keep their signatures). `DistanceMetric` becoming `#[non_exhaustive]` only affects *external* crates that exhaustively `match` on it ‚Äî they now need a wildcard arm; this is deliberate, made while still pre-1.0. The crate remains on a 0.x line ("expect refactors until v1.0").

## Verification

Run on Windows x86_64 (Rust stable 1.95.0). The same commands run in the CI matrix across Linux, macOS, and Windows at both `stable` and the `1.87` MSRV floor:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
cargo audit
```

All green. Test counts at this tag:

- **Default features:** 0 unit + 27 integration (`error_reason` 4, `properties` 9, `smoke` 9, `vector_id_display` 5) + 42 doctests = **69**.
- **`--all-features`:** 0 unit + 36 integration (`properties` rises to 17 ‚Äî the +8 are the serde round-trips; `smoke` 10) + 43 doctests = **79**.

`cargo deny check` reports `advisories ok, bans ok, licenses ok, sources ok`. `cargo audit` scans the dependency graph with zero advisories. `Vector::new` benches at ~10 ns (dim 32) to ~163 ns (dim 768), ~3.2‚Äì4.7 Gelem/s.

## What's next

- **v0.4.0 ‚Äî Feature freeze.** Finalize the surface against the first real consumers (`iqdb-distance` and an index crate compiling against it unchanged); declare the feature set frozen.
- **v0.5.0 ‚Äî Hardening + API freeze.** Final API review, cross-platform verification, and a conservative freeze of the public API for 1.x ‚Äî deliberate, because every other iQDB crate depends on it.
- **v0.6.0 ‚Üí v0.9.x ‚Äî Alpha / Beta / RC.** Integrate against real consumers, bug-fix, and polish toward 1.0.
