# iqdb-types v0.2.0 — Core Types

**The shared vocabulary lands.** v0.2.0 turns the v0.1.0 scaffold into the load-bearing type surface every other `iqdb-*` crate is built on: validated `f32` embeddings, stable identifiers, an immutable typed metadata map, a five-variant distance-metric tag, a boolean filter tree, search parameters, search hits, and the one domain error that ties them together. The crate is pure data — no engine, no storage, no I/O — and its only runtime dependency is `error-forge`. Optional `serde` derives ship behind a feature with zero cost when disabled.

## What is iqdb-types?

The root of the iQDB dependency graph. When you read the docs for `iqdb-hnsw`, `iqdb-flat`, or any other index crate, every type you meet — `Vector`, `VectorId`, `DistanceMetric`, `Hit`, `Metadata`, `SearchParams`, `Filter` — is defined here. It exists so the family can agree on shapes without depending on each other: a breaking change here cascades through every iQDB crate, so the surface is deliberately small, deliberately stable, and reviewed hard before it freezes at `1.0`. That caution is the crate's whole job.

## What's new in 0.2.0

### `Vector` / `VectorRef` — validated `f32` embeddings

`Vector` is the owned, dense `f32` embedding the family indexes. Validation happens at construction: empty inputs and any non-finite component (`NaN`, `+∞`, `−∞`) are rejected with `IqdbError::InvalidVector`. Downstream math never has to defend against invalid floats — every constructed `Vector` is known-good.

```rust
use iqdb_types::Vector;

// `new` validates contents up front (no empty, no NaN/Inf).
let v = Vector::new(vec![0.1, 0.2, 0.3])?;
assert_eq!(v.dim(), 3);
assert_eq!(v.as_slice(), &[0.1, 0.2, 0.3]);

// Fallible conversion for ergonomic `try_into()` call sites.
let w: Vector = vec![1.0, 2.0].try_into()?;
assert_eq!(w.dim(), 2);
# Ok::<(), iqdb_types::IqdbError>(())
```

`VectorRef<'a>` is the borrowed counterpart — a thin view over `&'a [f32]` for zero-copy passes through distance kernels. `Vector::new_unchecked` is a test-only escape hatch exposed exclusively by the `testing` feature; a production build cannot construct an invalid vector.

### `VectorId` — stable identifier

A `u64` or an opaque, non-empty byte key. `From<u64>` and `TryFrom<Vec<u8>>` (which rejects an empty key) cover construction from any source — autoincrement counters, hashed strings, snowflake ids — and `Display` gives a stable rendering for logs and diagnostics.

```rust
use iqdb_types::VectorId;

let a = VectorId::from(7u64);
assert_eq!(a, VectorId::U64(7));

let b = VectorId::try_from(vec![0xDE, 0xAD])?;          // non-empty byte key
assert!(matches!(b, VectorId::Bytes(_)));
# Ok::<(), iqdb_types::IqdbError>(())
```

### `Metadata` / `Value` — immutable typed metadata

`Metadata` is an immutable, ordered key/value bag travelling alongside a vector. It is `BTreeMap`-backed, so iteration order is deterministic — stable `serde` round-trips and stable test assertions across runs and machines. `Value` is a flat scalar: `String` / `Int(i64)` / `Float(f64)` / `Bool(bool)` / `Null`. There are no in-place mutators — you construct a new value.

```rust
use std::collections::BTreeMap;
use iqdb_types::{Metadata, Value};

let meta = Metadata::from_iter([
    ("source".to_string(), Value::String("wikipedia".into())),
    ("year".to_string(), Value::Int(2026)),
]);
assert_eq!(meta.get("year"), Some(&Value::Int(2026)));
assert_eq!(meta.len(), 2);
```

### `DistanceMetric` — the metric tag

Five metrics under one enum: `Cosine`, `DotProduct`, `Euclidean`, `Manhattan`, `Hamming`. This crate carries only the *tag* — the actual kernels (scalar reference paths plus runtime-dispatched AVX2/NEON) live in `iqdb-distance`. Keeping the tag here lets `SearchParams` and every index agree on the metric without pulling the math layer into the type crate.

### `Filter` — boolean expression tree over metadata

A composable predicate tree: `eq` / `neq` / `lt` / `lte` / `gt` / `gte` / `is_in` leaves combined with `and` / `or` / `not`. Filtering follows **closed-world** semantics: every leaf comparison on an *absent* field evaluates to `false`.

```rust
use iqdb_types::{Filter, Value};

// "published == true AND year >= 2020"
let f = Filter::and(vec![
    Filter::eq("published", Value::Bool(true)),
    Filter::gte("year", Value::Int(2020)),
]);
# let _ = f;
```

The closed-world rule has one sharp edge worth internalising: `Filter::neq("author", "ada")` does **not** match records that lack an `author` field. To express "anything that isn't `ada`, including records with no author," wrap an equality: `Filter::not(Filter::eq("author", "ada"))`. This is pinned in the rustdoc so the contract is unambiguous before consumers build on it.

### `SearchParams` / `Hit` — the query and its results

`SearchParams` carries `k`, the `DistanceMetric`, an optional search-breadth knob `ef` (e.g. HNSW `efSearch`), and an optional `Filter`. `SearchParams::new(k, metric)` covers the common path; struct-update syntax layers on the rest.

```rust
use iqdb_types::{DistanceMetric, Filter, SearchParams, Value};

let params = SearchParams {
    filter: Some(Filter::eq("published", Value::Bool(true))),
    ..SearchParams::new(10, DistanceMetric::Cosine)
};
assert_eq!(params.k, 10);
```

`Hit` is one result — a `VectorId`, an `f32` distance, and optional `Metadata` — with `Hit::new(id, distance)` for the metadata-free case.

### `IqdbError` / `Result` — the domain error

`IqdbError` implements `error_forge::ForgeError`, so every variant exposes a stable machine `kind()` and a distinct human-readable `caption()`, alongside `std::error::Error` and `Display`. Variants: `DimensionMismatch`, `InvalidVector`, `InvalidConfig`, `NotFound`, `Duplicate`, `InvalidMetric`, `InvalidFilter`, `ResourceLimitExceeded`. `Result<T>` is the crate-wide alias. Use `kind()` for branching; `caption()` for messages.

### Optional `serde` feature

Enabling `feature = "serde"` derives `Serialize` / `Deserialize` on every public type. `VectorRef` is `Serialize`-only — a borrowed view has nowhere to own decoded data. The default build pulls only `error-forge`; enabling `serde` adds the `serde` crate itself, while `serde_json` stays a dev-dependency used only by the round-trip tests.

### `VERSION`

`iqdb_types::VERSION` exposes the crate's compile-time `CARGO_PKG_VERSION` so consumers can report the exact `iqdb-types` build they link against — useful for diagnostics and version-skew checks across the family.

## Breaking changes

**Pre-1.0 API churn.** The crate is on a 0.x line; the stability promise is "expect refactors until v1.0." Relative to the v0.1.0 scaffold (which carried no domain logic), v0.2.0 is the first real surface — everything in *What's new* is an addition. `IqdbError` is `#[non_exhaustive]`, so consumers matching with a wildcard `_` arm absorb future variants without a code change.

```bash

cargo test
```

- **`--all-features`:** 0 unit + 19 integration + 43 doctests (the +1 integration and +1 doctest are the `serde` round-trip coverage).

## What's next

- **v0.3.0 — Ergonomics + property tests.** `serde` support and the fallible `Vector` construction surface already shipped early in 0.2.0; the outstanding 0.3 work is property-based coverage of the type contracts (ordering, equality, conversions, serde round-trips) and any remaining constructor ergonomics.
- **v0.4.0 — Feature freeze.** Finalize the surface against the first real consumers (`iqdb-distance` and an index crate compiling against it unchanged); add `examples/`.
- **v0.5.0 — Hardening + API freeze.** Final API review, cross-platform verification, complete `docs/API.md`. Because every other iQDB crate depends on this, the freeze is deliberate and conservative.
