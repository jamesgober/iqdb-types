# iqdb-types v1.0.0 — Stable

**The foundation is stable.** v1.0.0 commits the public API of `iqdb-types` under SemVer for the entire 1.x series: no breaking changes until 2.0. This is the bedrock the rest of the iQDB family is built on — the shared vocabulary every `iqdb-*` crate speaks — and it is now a fixed point they can depend on without fear of churn. Nothing changed in the code since 0.5.0; this release *is* the stability commitment.

## What is iqdb-types?

The root of the iQDB dependency graph: `Vector`, `VectorId`, `DistanceMetric`, `Metadata`/`Value`, `Filter`, `SearchParams`, `Hit`, and the `IqdbError` domain error. Pure data — no engine, no storage, no I/O — with a single runtime dependency (`error-forge`) and an optional `serde` feature.

```rust
use iqdb_types::{DistanceMetric, Filter, SearchParams, Value, Vector, VectorId};

let embedding = Vector::new(vec![0.1, 0.2, 0.3]).unwrap();
let id = VectorId::from(1u64);
let params = SearchParams {
    filter: Some(Filter::eq("published", Value::Bool(true))),
    ..SearchParams::new(10, DistanceMetric::Cosine)
};
assert_eq!((embedding.dim(), id, params.k), (3, VectorId::U64(1), 10));
```

## Why 1.0 now

The road here was deliberate, one verified phase at a time:

- **0.2.0 — Core Types.** The full type surface landed.
- **0.3.0 — Hardened & Documented.** Property tests for every invariant, runnable examples, a complete API reference, `Box<[f32]>`-backed `Vector`, an inlined surface, a benchmarked hot path, and `#[non_exhaustive]` where the surface should grow.
- **0.4.0 — Feature Freeze.** A consumer-simulation suite proved the vocabulary sufficient at the **exact signatures** the real `iqdb-distance`, index, and `iqdb-filter` crates expose (cross-checked against those implementations and their roadmaps).
- **0.5.0 — API Freeze.** The surface was locked and recorded; cross-platform verification on Windows + Linux across stable and the 1.87 MSRV.

By 0.5.0 every Definition-of-Done criterion (`dev/DIRECTIVES.md` §7) was met. The 0.6–0.9 RC phase exists to soak the API against live consumers; under the project's **types-first ordering** there are none yet, and its intent was already met by the consumer-simulation. So the crate proceeds to 1.0 on a fully-satisfied checklist rather than a calendar.

## The 1.x compatibility promise

- The public surface recorded in `dev/ROADMAP.md` is frozen until 2.0.
- Additive, non-breaking changes remain allowed within 1.x: new methods, and new variants on the `#[non_exhaustive]` `DistanceMetric` and `IqdbError`.
- `Value` and `Filter` are exhaustive and will not gain variants in 1.x (so consumers can match them fully and rely on it).
- One known integration step: a crate that exhaustively matches `DistanceMetric` (such as `iqdb-distance::dispatch`) must include a wildcard arm — the intended cost of a forward-compatible metric set.

## Breaking changes

**None.** No public API changed from 0.5.0; 1.0.0 is the SemVer stability commitment.

## Verification

Verified on Windows x86_64 **and** Linux x86_64 (WSL2 Ubuntu), each on stable (1.95.0) and the 1.87 MSRV (`cargo +1.87`). The CI matrix runs the same across Linux, macOS, and Windows at stable and 1.87:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
cargo audit
```

All green on every platform/toolchain combination. **90** tests on default features, **103** with `--all-features` (unit + integration + doctests), across happy-path, property-based, edge-case, and consumer-simulation suites. Zero `unsafe`. `cargo deny check` reports `advisories ok, bans ok, licenses ok, sources ok`; `cargo audit` finds zero advisories.

## What's next

`iqdb-types` is done. The family now builds on it: `iqdb-distance`, the index crates (`iqdb-flat`, `iqdb-hnsw`, `iqdb-ivf`), `iqdb-filter`, `iqdb-persist`, and the `iqdb` façade — each depending on this stable 1.x vocabulary. Anything those consumers surface that this crate should provide will be an additive `1.0.x` release.

## Installation

```toml
[dependencies]
iqdb-types = "1.0"

# Enable the optional `serde` feature
iqdb-types = { version = "1.0", features = ["serde"] }
```
